Realtime Continuous LOD Quadtree Terrain Rendering
Sept-2016

This project is an improvement to an earlier project I worked on rendering landscape using the OpenGL API. The older project draws the entire terrain in full detail. Increasing the terrain tile range will lower the frame rate because you are rendering more tiles and more triangles. On top of this detail in the distance can not be seen. It is a waste to have so many triangles drawn in the distance when those triangles are nearly as small as pixels. Level Of Detail (LOD) makes the terrain mesh dynamic, therefore the terrain tiles changes based on the observers' position. Terrain tiles closer to the viewer get rendered in full detail. As the distance between the viewer and terrain tile increases, fewer triangles are used to draw the terrain tiles. Because of this, the terrain tiles need to update whenever the viewer moves.

requirements: C/C++, OpengGL API 4.1, OpenGL Shading Language (GLSL), Xcode, Data Structures, 3D vector mathematics
source: code
